```python
class Solution:
    def findMaximumLength(self, nums: list[int]) -> int:
        n = len(nums)
        prefix = [0] * (n + 1)
        for i in range(n):
            prefix[i + 1] = prefix[i] + nums[i]
        last = 0
        pos = 0
        length = 0
        while pos < n:
            lo, hi = pos, n
            while lo < hi:
                mid = (lo + hi) // 2
                if prefix[mid + 1] - prefix[pos] >= last:
                    hi = mid
                else:
                    lo = mid + 1
            if lo == n:
                break
            last = prefix[lo + 1] - prefix[pos]
            pos = lo + 1
            length += 1
        return length
```

## Candidate Analysis

**Candidate 1: Greedy left-to-right with prefix sums and binary search**  
We process the array from left to right, maintaining the last value of the constructed nondecreasing sequence. For each new element, we try to extend the sequence by taking the smallest possible suffix sum that is ≥ last value. This can be done using prefix sums and binary searching for the earliest index where the sum from current position to that index meets the requirement. This is a known approach for similar "split array into nondecreasing sums" problems.  
Correctness: Greedy works because taking the smallest possible sum leaves more room for later elements.  
Complexity: O(n log n) with careful pointer management.

**Candidate 2: DP with monotonic queue**  
Define dp[i] = maximum length achievable for prefix ending at i, with last segment sum. Maintain a deque of candidate start positions where the segment sum is nondecreasing. This is more complex and may have O(n) amortized, but the greedy approach is simpler and sufficient.

## Selected Algorithm

Greedy left-to-right with prefix sums and binary search.

## State or Invariant

- `prefix[i]` = sum of nums[0..i-1] (size n+1, prefix[0]=0).
- `last` = sum of the last segment in the constructed nondecreasing sequence.
- `pos` = current index in nums (0-based) where we are building the next segment.
- Invariant: We always extend the sequence by the shortest possible segment starting at `pos` whose sum ≥ `last`. This minimizes the last sum and maximizes future flexibility.

## Complexity

- Time: O(n log n) — each element is processed once, and for each we may binary search over the remaining suffix (O(log n)).
- Space: O(n) for prefix sums array.
