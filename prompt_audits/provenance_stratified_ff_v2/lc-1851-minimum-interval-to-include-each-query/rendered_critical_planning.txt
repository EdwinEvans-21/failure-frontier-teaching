class Solution:
    def minInterval(self, intervals: list[list[int]], queries: list[int]) -> list[int]:
        intervals.sort(key=lambda x: x[1] - x[0] + 1)
        sorted_queries = sorted([(q, i) for i, q in enumerate(queries)])
        res = [-1] * len(queries)
        heap = []
        idx = 0
        n = len(intervals)
        for q, orig_idx in sorted_queries:
            while idx < n and intervals[idx][0] <= q:
                l, r = intervals[idx]
                heapq.heappush(heap, (r, r - l + 1))
                idx += 1
            while heap and heap[0][0] < q:
                heapq.heappop(heap)
            if heap:
                res[orig_idx] = heap[0][1]
            else:
                res[orig_idx] = -1
        return res

## Candidate Analysis

**Candidate 1: Sort intervals by length, process queries sorted, use a min-heap of active intervals**  
- Sort intervals by length ascending. Sort queries with original indices.  
- For each query (in sorted order), add all intervals whose left endpoint ≤ query into a min-heap keyed by right endpoint.  
- Pop from heap until top interval's right endpoint ≥ query; if heap empty, answer is -1, else answer is length of top interval.  
- This works because we process queries in increasing order, so intervals once added remain candidates for later queries. The heap always gives the shortest interval that covers the current query among those added so far.  
- Complexity O((N+Q) log N). Correct.

**Candidate 2: Sweep line over all points, maintain a multiset of interval lengths**  
- Collect all interval start and end+1 events. Sort events. Sweep from left to right.  
- At a start event, add interval length to a multiset. At an end+1 event, remove that length.  
- For each query, binary search its position in the event list and answer the minimum length in the multiset at that point.  
- Problem: The multiset state is only defined at event points; between events it's constant, but queries may fall between events. This can be handled by processing queries sorted along with events. However, maintaining a multiset that supports removal of arbitrary lengths (not just min) requires a data structure like a balanced BST or two heaps with lazy deletion.  
- Complexity O((N+Q) log N). Also correct, but more complex to implement than Candidate 1.

**Rejection of Candidate 2**: While correct, Candidate 1 is simpler and more standard for this problem (LeetCode 1851). Candidate 2 requires careful handling of removal and event ordering. Candidate 1 is preferred.

For each query, add all intervals whose left ≤ query, then pop from heap until top interval's right ≥ query. Answer is length of top interval or -1.
