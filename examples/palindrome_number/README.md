# Palindrome Number

This is the easy sanity/control problem for the Failure-Frontier Teaching study. It is intended to validate the judging pipeline and provide straightforward successful samples, not to produce a high rate of teacher failures.

Implement `Solution.isPalindrome(x)`. The input is one signed 32-bit integer. Return a Python `bool` indicating whether its decimal representation is unchanged when read in reverse. Returning an integer such as `0` or `1` does not satisfy the output contract.

The public cases contain only a few representative examples. Hidden cases cover the documented input categories and boundaries without exposing their concrete values through public descriptions or model feedback.

Source: [LeetCode 9 - Palindrome Number](https://leetcode.com/problems/palindrome-number/). This summary is written specifically for this project and does not reproduce the original problem statement.
